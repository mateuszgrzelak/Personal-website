I am a automatic control and robotics engineer who's knowledgeable in both computer science and electrical engineering. I decided to change career path and become a programmer, because programming gives me the greatest satisfaction. Below you can see some of rules that I follow:
1. Try to apply design patterns whenever possible.
Design patterns
2. Leave your code better than you found it.
Boy Scouts have a rule regarding camping, that they should leave the campground cleaner than they found it.
3. Test first
Tests have a lot of adventages when you know how to write them. Good written tests can save a lot o time and nerves. I try to use TDD as often as possible.
4. Create readable programs for people, not just for computers.
Code should be written in that way, that every programmer should read it without problems. This means that each variable name should clearly indicate what it is for.
